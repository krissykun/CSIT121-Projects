Web Standards: Payment Method Identifiers (PMI)

What are PMIs?

Think of them as a sort of "fingerprint" for payment methods. Just like a fingerprint is unique to each person, a PMI is unique to each payment method.

Payment Method Identifiers (PMIs) are unique identifiers used to represent different payment methods in a standardized way. They are used by payment processors, banks, and other financial institutions to ensure that payments are processed accurately and securely.

They can be found in two forms: URL-based PMIs or standardized PMIs. Each is covered in more detail below:

url-based PMIs

URL-based PMIs are formatted as a typical URL, which makes them easy to use in web apps and APIs. For example, a URL-based PMI for a service like Apple Pay might look like this: https://apple.com/apple-pay

Steps for validating a url-based PMI:

  1. If url's scheme is not "https", invalid
  2. If url's username or password is not the empty string, invalid
  3. Otherwise, valid

The scheme is the part of the URL that indicates the protocol used to access the resource. In this case, it should be "https" to ensure a secure connection.

Examples:

Valid:

  • https://example.com/pay
  • https://example.com/pay/version/1

Invalid:

  • http://username:password@example; uses http://, a username and a password
  • unknown://example.com/pay; uses an unknown scheme

Standardized PMIs

Standardized PMIs take the form of a string that represents a standardized payment method. For example, a standardized PMI for a payment method like Visa might look like this: visa-card

Steps for validating a standardized PMI:

  1. If string conforms to the syntax of a standardaized PMI, valid
  2. Otherwise, invalid

The syntax of a standardized PMI is given by the following (luckily, users won't need to think about it in this way):

stdpmi = part *( "-" part )
part = 1loweralpha *( DIGIT / loweralpha )
loweralpha = %x61-7A
